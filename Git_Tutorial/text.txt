    # Student Management System

    A simple Python-based Student Management System that allows users to add, view, search, and delete student records using file handling and dictionaries.

    ---

    ## Features

    * Add student details
    * View all students
    * Search student by ID
    * Delete student record
    * Simple command-line interface
    * File handling support

    ---

    ## Tech Stack

    * Python 3

    ---

    ## Project Structure

    ```text
    student-management-system/
    │
    ├── data/
    │   └── students.json
    │
    ├── src/
    │   ├── main.py
    │   ├── operations.py
    │   └── utils.py
    │
    ├── README.md
    ├── requirements.txt
    └── .gitignore
    ```

    # Collaborative Workflow

    This project follows a feature-branch collaborative workflow using Git and GitHub.

    ---

    ## Workflow Steps

    ### 1. Clone the Repository

    ```bash
    git clone https://github.com/your-username/project-name.git
    ```

    ---

    ### 2. Move to Project Directory

    ```bash
    cd project-name
    ```

    ---

    ### 3. Create a New Feature Branch

    ```bash
    git checkout -b feature/your-feature-name
    ```

    Example:

    ```bash
    git checkout -b feature/login-system
    ```

    ---

    ### 4. Make Changes

    Develop your feature or bug fix inside the new branch.

    ---

    ### 5. Commit Changes

    ```bash
    git add .
    git commit -m "added login functionality"
    ```

    ---

    ### 6. Push Branch to GitHub

    ```bash
    git push origin feature/login-system
    ```

    ---

    ### 7. Create Pull Request

    * Open the repository on GitHub
    * Click **Compare & pull request**
    * Add PR title and description
    * Submit the Pull Request

    ---

    ### 8. Code Review Process

    Team members review the code and may:

    * approve changes
    * request modifications
    * suggest improvements

    ---

    ### 9. Update Pull Request (if needed)

    After making requested changes:

    ```bash
    git add .
    git commit -m "fixed review comments"
    git push
    ```

    The Pull Request updates automatically.

    ---

    ### 10. Merge Pull Request

    Once approved:

    * Click **Merge Pull Request**
    * Merge changes into `main`

    ---

    ### 11. Delete Feature Branch

    ```bash
    git branch -d feature/login-system
    git push origin --delete feature/login-system
    ```

    ---

    ## Best Practices

    * Never work directly on `main`
    * Pull latest changes regularly
    * Write meaningful commit messages
    * Keep Pull Requests small and focused
    * Review code before merging

    ---

    ## Branch Naming Convention

    Examples:

    ```text
    feature/login-system
    bugfix/navbar-error
    docs/readme-update
    hotfix/payment-issue
    ```
